Introdução a HTML

voltar ao índice

O que é HTML? HTML é uma linguagem usada para criar páginas de internet. Ele é como se fosse a “estrutura” de um site, igual o esqueleto do nosso corpo. Com o HTML, a gente diz onde vai o título, o texto, as imagens e os botões de uma página. Ele funciona com “palavrinhas” especiais chamadas tags, que mostram ao computador o que cada coisa é. Então, quando você entra em um site e vê palavras, imagens e botões, pode ter certeza: por trás disso tudo, tem HTML ajudando a organizar tudo direitinho!

Trabalhando com tabelas HTML

voltar ao índice

Uma tabela no HTML serve para organizar informações em linhas e colunas, como uma grade que ajuda a deixar tudo bem arrumadinho. Imagine uma tabela parecida com aquela que usamos na escola, onde as coisas ficam alinhadas e fáceis de entender. Cada tabela tem várias linhas, que são como fileiras horizontais, e várias colunas, que são as colunas verticais. O espaço onde uma linha e uma coluna se encontram chama-se célula, e é ali que colocamos os dados, como nomes, números ou palavras. As tabelas são muito úteis quando queremos mostrar listas de preços, horários de ônibus, resultados de jogos ou qualquer coisa que precise estar bem organizada e fácil de comparar. Elas ajudam quem visita a página a encontrar a informação de forma rápida. Normalmente, as tabelas têm uma primeira linha chamada cabeçalho, que mostra o nome de cada coluna, como “Produto”, “Preço” ou “Quantidade”. Isso ajuda a entender melhor o que cada informação significa. Além de deixar os dados organizados, as tabelas podem ser estilizadas com cores, bordas e espaçamentos para ficar mais bonitas e fáceis de ler. Isso é feito com outras linguagens, mas o importante é que a estrutura da tabela é sempre a mesma: linhas, colunas e células. Mesmo sendo muito úteis, as tabelas devem ser usadas só para mostrar dados, não para montar o visual completo de uma página. Para isso, existem outras ferramentas melhores. No geral, uma tabela em HTML é uma forma simples e prática de mostrar informações arrumadas, para que qualquer pessoa que visite o site consiga entender e comparar os dados sem dificuldade.

Estruturando seu HTML + Formatação

voltar ao índice

Quando começamos a criar uma página na internet usando HTML, é importante saber como estruturar e organizar o código para que ele funcione bem e seja fácil de entender. Estruturar o código significa colocar as partes certas no lugar certo, seguindo uma ordem que o navegador entende. Toda página HTML começa com uma estrutura básica que mostra onde começa e termina o código. Essa estrutura tem algumas partes principais: uma parte que diz que o documento é HTML, uma parte que guarda informações que não aparecem na página, e outra que tem tudo o que vai aparecer para quem visitar o site. Para deixar o código organizado, usamos “tags”, que são palavras especiais entre sinais de menor e maior, como " <" e ">" . Cada tag tem uma função, por exemplo, para criar títulos, parágrafos, listas, imagens e muito mais. É importante abrir a tag e fechá-la para o navegador entender onde começa e termina cada parte. Além disso, formatar o código significa deixar ele arrumadinho, com espaços e quebras de linha para facilitar a leitura. Assim, fica mais fácil para quem vai olhar o código depois, seja você mesmo ou outra pessoa, entender o que está acontecendo. Um jeito fácil de formatar o código é usar a indentação, que é quando a gente deixa um pedaço do código um pouco para dentro, para mostrar que ele está dentro de outra parte. Por exemplo, o conteúdo dentro de uma tag vai ficar com um espaço a mais que a tag que o envolve. Ferramentas como o Visual Studio Code ajudam muito nisso, pois têm comandos que organizam o código automaticamente, deixando tudo bonitinho com um só comando. Manter o código organizado e bem estruturado facilita a manutenção da página, ajuda a encontrar erros e torna o trabalho mais profissional, mesmo quando a gente está aprendendo.

Trabalhando com formulário simples

voltar ao índice

Os formulários em HTML são usados para receber informações das pessoas que visitam um site. Eles funcionam como uma espécie de “caixa” onde o usuário pode escrever textos, escolher opções, marcar coisas e enviar esses dados para o site. Um formulário pode ter vários tipos de campos, como espaços para digitar o nome, o e-mail, senhas, escolher datas, marcar caixas de seleção ou escolher entre várias opções. Tudo isso ajuda a coletar informações importantes. Quando a pessoa preenche o formulário e clica em um botão para enviar, essas informações podem ser usadas para várias coisas, como fazer um cadastro, entrar em uma conta, enviar uma mensagem ou até fazer compras online. Criar um formulário no HTML é simples, pois existem tags específicas para isso, mas o mais importante é pensar em deixar ele fácil de usar, para que quem estiver usando o site consiga preencher e enviar sem problemas. Além disso, os formulários podem ser combinados com outras ferramentas para garantir que as informações sejam enviadas corretamente e que o site possa responder de forma rápida. No geral, os formulários são uma parte muito importante dos sites, porque ajudam a transformar visitantes em participantes, clientes ou usuários, permitindo que o site receba informações úteis e interaja com as pessoas.

Trabalhando com midias utilizando HTML

voltar ao índice

No HTML, trabalhar com mídias significa adicionar imagens, vídeos, sons e outros arquivos para deixar a página mais interessante e interativa. As mídias ajudam a explicar melhor as informações e deixam o site mais bonito e atraente para quem visita. Para colocar uma imagem na página, usamos um comando especial que diz onde está o arquivo da imagem e mostra ela para o visitante. Assim, podemos colocar fotos, desenhos ou qualquer figura que ajude a contar uma história ou ilustrar um conteúdo. Também podemos adicionar vídeos, que são arquivos que mostram movimentos e sons juntos. Eles podem ser vídeos explicativos, músicas, apresentações ou qualquer coisa que queremos mostrar para o usuário. Os sons, como músicas ou efeitos, também podem ser incluídos para tornar a experiência no site mais completa. Eles podem tocar automaticamente ou quando a pessoa clicar em algum lugar. É importante lembrar que, ao usar mídias, precisamos cuidar do tamanho dos arquivos para que o site carregue rápido e funcione bem para todo mundo, até para quem tem internet mais lenta. No geral, usar mídias no HTML é uma forma simples de deixar as páginas mais vivas e ajudar o visitante a entender melhor o conteúdo que está sendo mostrado.

Uso da semântica no HTML

voltar ao índice

No HTML, a semântica significa usar as tags corretas para dar significado ao conteúdo da página. Em vez de usar apenas tags genéricas, a gente escolhe aquelas que mostram exatamente o que cada parte representa, como um título, um parágrafo, uma seção ou um rodapé. Usar semântica ajuda o navegador e outros programas, como leitores de tela para pessoas com deficiência, a entender melhor o que está escrito na página. Isso faz com que o site fique mais acessível e organizado. Além disso, a semântica facilita o trabalho dos motores de busca, como o Google, que conseguem entender melhor o conteúdo do site e, assim, mostrar ele para mais pessoas. Quando a gente escreve um código semântico, também fica mais fácil para os desenvolvedores entenderem e manterem o site, porque o código é mais claro e organizado. Em resumo, usar semântica no HTML torna a página mais acessível, melhora o posicionamento nos motores de busca e deixa o código mais fácil de entender e cuidar.

Principais tags HTML

voltar ao índice

No HTML, existem várias tags principais que usamos para montar uma página da web. Cada uma tem uma função diferente e ajuda a deixar o conteúdo bem organizado. A primeira de todas é a tag "html", que envolve todo o conteúdo da página. Ela diz para o navegador que aquilo é um documento HTML. Dentro dela, a gente tem duas partes importantes: a tag "head" e a tag "body". A "head" guarda informações que não aparecem direto na tela, como o título da aba e links de configuração. Já a "body" é onde colocamos tudo o que o visitante vai ver no site, como textos, imagens, vídeos, listas, etc. Para colocar títulos, usamos as tags "h1" até "h6". A "h1" é o título mais importante e maior, e a "h6" é o menor. Isso ajuda a dividir o conteúdo e deixar a leitura mais organizada. Os parágrafos são feitos com a tag "p", que é usada para escrever blocos de texto. Se quiser destacar alguma palavra, dá pra usar "strong" para negrito e "em" para itálico. Quando queremos listar coisas, usamos as tags de listas. A "ul" cria uma lista com bolinhas, como uma lista de compras. A "ol" cria uma lista numerada, tipo uma ordem de tarefas. Cada item da lista vai dentro de uma tag chamada "li". Para adicionar links, usamos a tag "a", que transforma um texto em algo clicável. Assim, a pessoa pode clicar e ir para outro site ou página. As imagens entram com a tag "img". Com ela, você mostra uma foto ou imagem no site. É só dizer onde está o arquivo da imagem e colocar uma descrição para acessibilidade. Se quiser mostrar informações em colunas e linhas, usamos tabelas. A tag "table" monta a estrutura da tabela. Cada linha é feita com a tag "tr", os títulos das colunas com "th" e os dados com "td". Para fazer formulários, que são aqueles campos onde a pessoa preenche nome, e-mail, senha e clica em enviar, usamos a tag "form". Dentro dela, dá pra colocar vários tipos de campos e botões. Também tem a "div", que é usada para agrupar partes do site, como se fossem caixinhas de conteúdo. Já a "span" serve para mexer em pedaços menores de texto. Por último, tem as tags chamadas semânticas, como "header", "nav", "section", "article" e "footer". Essas ajudam a dar significado para cada parte do site, tipo o cabeçalho, o menu, as seções de conteúdo, artigos e o rodapé Essas são as principais tags que você vai usar no HTML. Com elas, dá pra montar páginas bem organizadas e mais fáceis de entender, tanto para quem vê quanto para quem cuida do código.
